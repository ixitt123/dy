# 微信朋友圈文案与表情

生成像真人分享的朋友圈文案：事实真实、语气温和、宣传信息清楚但不硬广。表情服务阅读节奏、场景和情绪，不是装饰品。

## 工作流

1. 保留原文中的事实、对象、活动信息和承诺边界；没有依据时不补案例、数据、效果或紧迫感。
2. 用“场景或观察 → 自然判断 → 具体价值 → 低压力行动邀请”组织正文。短句、自然换行，避免公告腔、AI 套话和强推销。
3. 输出可直接复制发布的排版：不加标题、编号、Markdown、解释或“文案如下”；按篇幅组织 2-6 个短段落，每段只承载一个意思，段落间留一个空行。使用自然中文标点，清理多余空格、连续空行和句末重复符号；不要把每一句都强行换行。
4. 根据表情模式处理正文：
   - `no`：不使用 emoji、颜文字或表情包符号。
   - `yes`：模型不要临时生成 emoji，只负责写出适合插入表情的句子、停顿和段落。运行时清除模型表情，再由代码预制表情库根据主题、篇幅和语义统一选择：普通篇幅通常 2–3 个，长文、多个场景或情绪转折较多时自然增加。
5. 逐段检查表情效果：不要连续堆叠、不要每句都加、不要用无关或过度兴奋的表情。删除任何删掉后文意不受影响、只是在凑数的表情。
6. 输出前确认：正文读起来仍像一个人在说话；表情与上下文匹配；引用素材和行动建议都自然融入正文。

## 原文直用模式

当页面选择“原文生成朋友圈文案+图片提示词”时：

1. 朋友圈正文不得经过改写、扩写、删减、字数校准、人设重塑或引用素材注入。
2. 只允许整理空格和自然段换行；原文字词、顺序、标点和已有内容必须保留。
3. `yes` 模式可由代码预制表情库在合适位置插入表情，`no` 模式不新增表情；两种模式都不得删除原文已有内容。
4. 图片提示词只根据整理后的原文和用户选择的图片 Skill、配图数量等视觉设置生成，不得引入引用素材中的名言、热梗、金句、古诗、案例、数据或观点。
5. 整理后的原文直接写入“朋友圈文案成品”区域，图片提示词继续写入配图提示词区域。

## 表情选择原则

- 生活观察或轻松提醒：可用 `🙂`、`👌`、`✨`，但先看语境。
- 经验、认可或收获：可用 `✅`、`📌`、`🌿`，避免连续使用同类符号。
- 行动邀请或信息提示：可用 `📝`、`📩`、`👋`，不制造稀缺和焦虑。
- 避免 `🔥🚀💥` 等夸张营销感表情，除非原文语气本来就强烈且确有必要。

## 质量底线

- 不因为表情而改变事实或夸大价值。
- 不把表情放在标题、每段开头或句末成串。
- 表情可以多于 3 个，但必须随篇幅和内容自然增长；宁可少而准，不要为了数字堆砌。
- 交付的 `post` 字段只放可发布正文，保留自然段，不附加“生成说明”“排版建议”或图片提示词。
