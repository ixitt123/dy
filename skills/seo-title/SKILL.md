# SEO Title

## 输入

- TTS 原始文案或清洗后的口播文案。
- 可选的视频类型、平台、项目标题或用户手动标题。
- 可选的字幕时间轴，用于校验标题是否偏题。

## 输出

- `title`：默认发布标题，优先给抖音和生产线使用。
- `platformTitles`：抖音、小红书、视频号、B 站的标题。
- `seoKeywords`：从文案中提取的搜索关键词。
- `hashtags`：平台发布标签。
- `titleScore`：标题长度、关键词命中、清晰度和风险评分。

## 规则

1. 标题必须来自文案真实主题，不编造学校、课程、承诺、成绩、收益或案例。
2. 标题必须包含至少一个可搜索关键词，例如行业、对象、问题、方法、平台或场景。
3. 默认标题控制在 8-28 个中文字符，适合短视频列表展示。
4. 小红书标题可以更偏经验感；视频号标题更稳；B 站标题可更说明性。
5. 禁止使用 `保证`、`百分百`、`包过`、`稳赚`、`绝对` 等夸大承诺。
6. 标题要随 TTS 三件套一起传递：音频、文案、带时间戳字幕必须绑定同一个标题。

## 生产线契约

发送 TTS 到生产线时，payload 必须包含：

- `title`
- `seo_title`
- `publish_title`
- `platform_titles`
- `seo_keywords`
- `hashtags`
- `title_score`

下游模块不得再用音色名、文件名或文案前几个字临时充当项目标题。
